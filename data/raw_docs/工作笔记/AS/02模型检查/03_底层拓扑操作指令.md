这份文档介绍的是 Analysis Situs (AS) 中非常实用且直观的底层拓扑操作指令：**`explode`（炸开/分解）**。

在 B-rep（边界表示法）的层级结构中，这个命令相当于一把“解剖刀”，可以帮你把复杂的零件一层层剥开，或者直接提取出你想要的特定底层元素。

---

### 一、 核心功能与应用场景

- **本质动作：** 将当前激活的零件（Active part）拆解为构成它的子形状（Subshapes）。
    
- **核心用途：**
    
    1. **拓扑结构审查：** 肉眼检查一个复杂 B-rep 模型底层的拓扑嵌套关系是否正确。
        
    2. **几何元素隔离：** 从庞大的装配体或实体中，单独抽离出特定的面或边，以便送入下游算法（如特征识别、网格划分）进行单独分析。
        

### 二、 核心执行逻辑：逐级剥离 vs 跨级提取

`explode` 命令的执行逻辑完全取决于你是否赋予了它特定参数。这对应了拓扑图（Topology Graph）的两种遍历方式：

#### 1. 默认行为：逐级剥离（不带任何参数）

如果不带参数直接运行 `explode`，它只会把模型**降解到它的“直系下一代”**（Direct children），绝不越级。

- **Compound (复合体)** $\rightarrow$ 炸成它包含的各个独立实体 (Child shapes)。
    
- **Solid (实体)** $\rightarrow$ 炸成包裹它的 **Shells (壳)**。
    
- **Shell (壳)** $\rightarrow$ 炸成组成它的 **Faces (面)**。
    
- **Face (面)** $\rightarrow$ 炸成它的边界 **Wires (线框)**。
    
- **Wire (线框)** $\rightarrow$ 炸成具体的 **Edges (边)**。
    

#### 2. 目标提取：跨层级穿透（带拓扑限制参数）

如果你明确知道自己想要什么，可以使用“拓扑限定符（Subshape qualifiers）”。算法会直接穿透层级，把整个模型里所有符合该类型的元素全抓出来：

- `-solid`：提取所有实体。
    
- `-shell`：提取所有壳。
    
- `-face`：提取所有面（最常用，比如直接把一个装配体炸成几千个离散的面）。
    
- `-wire`：提取所有线框。
    
- `-edge`：提取所有边。
    
- `-vertex`：提取所有顶点。
    

#### 3. 特殊空间位置参数

- `-noloc`：**忽略空间累积变换。** 只炸开到直系子代，但是不附带它们在装配体中累积的坐标位置（Locations）。_注意：该参数不能和上面的拓扑限定符混用。_
    

### 三、 命令的“副作用”与工程细节

执行该命令后，除了在底层数据结构上进行了拆解，还会触发以下系统级联动（Side effects）：

1. **返回值 (Return Value)：** 命令在执行完毕后，会返回被炸出的子形状的**总数量**。
    
2. **UI 渲染 (Imperative Viewer)：** 炸开后的所有碎片，会立即在命令式视图（Imperative Viewer）中被单独绘制出来。
    
3. **对象树更新 (Object Browser)：** 这些碎片会被作为独立的节点，追加到对象浏览器中，方便你随后进行单独点击和排查。
    
4. **显示属性继承：** 炸出来的子形状，会自动继承其父级零件的网格细分属性（线性偏离度和角度偏离度，即显示精度），保证视觉上的一致性。
    

### 四、 Tcl 自动化脚本实战套路

在批处理脚本中，`explode` 经常与变量捕获结合使用，用于统计和遍历：

**套路 1：提取全模型所有的面，并统计数量**

```
# 加载 STEP 装配体文件
load-step assembly.stp
# 跨层级炸开所有面，并将返回的面数量存入变量 numFaces
set numFaces [explode -face]
puts "成功提取了 $numFaces 个面！"
```

**套路 2：降级剥离**

```
# 加载零件并仅剥离到下一层级
load-brep mypart.brep
explode
```

### 五、 关联诊断命令

- `print-summary`：不执行炸开操作，仅仅在控制台打印当前激活零件的内部子形状统计清单。
    
- `get-summary`：静默获取子形状的统计数量，并输出到 Tcl 变量中供脚本调用。
