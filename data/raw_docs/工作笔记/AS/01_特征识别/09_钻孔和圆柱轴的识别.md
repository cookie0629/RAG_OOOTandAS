### 一、 核心目标与工业价值

提取钻孔特征不仅是为了“认出它”，更是为了下游环节服务：

- **制造规划 (CAM)：** 决定选用多大的钻头、走什么样的刀路。
    
- **成本估算：** 根据孔的数量、深度和复杂度计算加工耗时。
    
- **模型简化 (Defeaturing)：** 在做 CAE 仿真分析前，一键填平这些对力学影响不大的小孔以减少计算量。

### 二、 早期思路：参数域识别 (Parametric Domain) —— 为什么被淘汰？

最初直觉的想法是“降维打击”：如果一个圆柱形钻头在一个平面上打孔，那么在这个面的 2D UV 参数空间里，一定会留下一个**圆形轮廓线 (2D Curve)**。只要找到这些圆圈，量出直径不就行了吗？

**致命缺陷（极易翻车的四个原因）：**

1. **数据不纯粹：** 底层几何引擎在计算面与面相交时，有时会生成“样条曲线 (Spline)”而不是完美的标准“圆”。如果算法只死板地寻找圆，就会漏判。
    
2. **精度丢失：** 就算用了曲线拟合算法强制找圆，由于计算误差，原本 6.0mm 的孔可能会被拟合出 6.1mm 这种非标准尺寸，导致加工干涉。
    
3. **相交面非平面：** 如果你是在一根圆管上打孔，圆柱切圆柱，交线根本不是圆，而是一条极其复杂的**斯坦梅茨曲线 (Steinmetz curve)**。
    
4. **无法识别复合特征：** 这种依靠 2D 边缘线的算法，根本无法把沉头孔的锥面和内部圆柱面联系起来。
    

**剩余价值：**

目前这种基于 2D 轮廓线的识别仅在**钣金加工**中有用武之地（仅用于快速区分简单的切口和通孔）。

### 三、 终极解法：基于曲面识别 (Surface-based Recognition)

AS 最终采用的 Pragmatic (务实) 方案，是直接从 **3D 曲面 (Faces)** 入手，而非边缘线。因为钻头切削留下的曲面（如完美的圆柱面、圆锥面），其底层的数学属性是被完整保留的。

识别过程分为严密的“两步走”战略：

#### 第一步：找特征嫌疑人 (局部提取)

不急于定性，先通过几何对称性，把所有“看起来像孔”的面组提取出来孤立分析。

- 寻找**同轴的圆柱面**（这是沉孔的标志）。
    
- 寻找**同轴的圆锥面与圆柱面组合**（这是沉头孔的标志）。
    

#### 第二步：查字典对暗号 (模式匹配与图同构)

这是最核心的算法落地。AS 内部维护了一个**特征字典 (JSON 格式)**。

|**字典关键字段**|**作用解析**|
|---|---|
|`type`|孔的最终定性（如 `countersunk-hole` 沉头孔）。|
|`binbrep`|这是一个经过 base64 编码的**真实 CAD 微型模型块**（标准特征的几何样本）。|
|`sinkFaces` / `holeFaces`|提前打好的语义标签。指明在这个标准样本中，哪个面是“沉头锥面”，哪个面是“直孔面”。|

**执行逻辑：**

系统会将刚才提取出来的嫌疑面组，转化为 AAG（属性邻接图），然后去跟字典里 `binbrep` 样本的 AAG 进行**图同构匹配 (Subgraph Isomorphism)**。一旦图谱匹配成功，算法就能直接根据字典里的标签，准确地给模型上的面赋予属性（“这个面是沉头区，那个面是钻孔区”），从而完美提取出盲孔、通孔、复合孔的所有加工参数。



### 一、 核心逻辑：“轴是反转的孔”

在算法视角下，圆柱轴（Shaft）被定义为**“向外反转的孔”**（A hole inverted）。

因此，识别圆柱轴的底层逻辑与识别普通钻孔高度一致，都采用了 **“识别器/规则 (Recognizer/Rule) 架构”**。它依赖拓扑关系（如二面角的凸性，凸台周围通常是凸边）和几何特征（圆柱面）来进行规则过滤。

### 二、 算法流程拆解（基于伪代码的工程解析）

AS 在底层使用了一个非常经典的遍历与图论结合的算法模板。我们可以将文档中的伪代码翻译为以下四个标准步骤：

1. **游标遍历 (Cursor Iteration)：**
    
    算法拿到整个 CAD 模型的面集合（$F$），开始进行全局 `for` 循环遍历。使用一个 `visited` 集合来记录已经处理过的面，避免重复计算。
    
2. **规则蔓延 (Rule Application)：**
    
    当游标遇到一个尚未访问的面时，触发 `ApplyRule(f)`。这个 Rule 就像一个探照灯，如果当前面符合“轴”的初始特征（例如是一个凸圆柱面），它会顺着 AAG 拓扑图向外蔓延，把相邻的、同属这个轴的面全部找出，并一并加入 `visited` 集合。
    
3. **提取连通分支 (Connected Component Extraction)：**
    
    在 AAG 图 $G$ 中，刚才被 Rule 成功标记的那些面，会天然形成一个个独立的局部网络（即图论中的“连通分支” $c$）。每一个连通分支就代表一个独立的“物理轴”。
    
4. **属性解析与封装 (Property Population)：**
    
    调用 `PopulateFeature(c)`，针对每一个连通分支进行计算，提取出这个轴的工程属性，最终存入结果集 $R$ 中。
    

### 三、 C++ 接口层与输出结果的高级特性

这部分代码展示了 AS SDK 在工程落地时的一个优秀设计原则：**不仅输出索引，更要输出业务语义。**

- **输入控制：** 实例化 `asiAlgo_RecognizeShafts` 时传入 AAG 对象，在调用 `Perform(r)` 时，允许传入一个最大半径 `r`。这在实际业务中非常有用（例如：只识别半径小于 50mm 的轴，忽略巨大的圆柱形主壳体）。
    
- **两层数据输出：**
    
    - **底层拓扑数据：** 通过 `GetResultIndices()` 可以获取所有构成轴的面 ID 集合（方便后续做高亮显示或拓扑修改）。
        
    - **高层业务数据：** 通过 `GetShafts()` 返回的是一个被封装好的 `shaft` 对象数组。系统已经自动根据拓扑连通性将面进行了分组，并计算出了该轴的具体 **直径 (diameter)** 和 **长度 (length)**。

### 思考批注

1. **AAG 图论在规则法中的降维作用：** 许多传统 CAD 软件或开源脚本在识别圆柱时，往往找出一堆散乱的圆柱面，需要开发者自己写复杂的逻辑去判断哪些面属于同一个轴。而 AS 的优势在于利用 AAG 连通图（Connected Component），**天然地在底层完成了特征的分组**，极大降低了上层业务代码的复杂度。
    
2. **算法健壮性测试思路：**
    
    在用控制变量法对比 AS 与 OCCT 原生能力、或与其他友商产品（如 ANSA, 西门子）对比时，可以使用**“复合阶梯轴（多个不同直径圆柱面相连）”**、**“带有键槽的轴（圆柱面被破坏）”**的测试文件，来测试各家软件的 Rule 规则是否足够强壮，能否精准提取出整个阶梯轴的长度，或者会不会因为键槽的存在导致识别中断。
