**绿色区块代表开源组件 (OPEN SOURCE)**，**蓝色区块代表商业组件 (COMMERCIAL)**

架构图各层级和模块的详细总结：

### 1. 数据输入层 (Inputs)

系统位于架构图的左侧接收外部数据。支持的输入格式非常广泛，包括：

- **标准 CAD 格式：** STEP, BREP, IGES 等。
    
- **网格格式：** STL。
    
- **专有格式：** Proprietary formats（未具体列出，通常指其他商业 CAD 软件的闭源格式）。
    

### 2. 核心数据总线 (Core Hub)

- **数据交换接口 (Data Exchange Interfaces)：** 这是一个横贯系统的核心枢纽（绿色开源）。它接收输入数据，并与上方的“装配体 (Assemblies)”、下方的各个处理模块进行双向的数据交互，最后将处理结果传递给输出端。
    
- **装配体 (Assemblies)：** 直接与数据交换接口相连，用于处理和管理多部件的复杂装配体结构。
    

### 3. 核心处理模块 (Core Processing Modules)

这是该内核的开源“计算大脑”，主要分为两大并行分支：

- **特征识别 (Feature Recognition)：** 用于从基础几何体中识别出具有工程意义的特征（如孔、槽等）。包含：
    
    - 底层架构 (Base architecture)
        
    - AAG (Attributed Adjacency Graph，属性邻接图，特征识别的核心数据结构)
        
    - 同构 (Isomorphism，常用于图匹配识别)
        
    - 基础识别器 (Basic recognizers)
        
- **建模 (Modeling)：** 提供了一套完整的几何处理与修复工具链，包含：
    
    - 检查 (Inspection)
        
    - 几何修复 (Healing)
        
    - 曲面建模 (Surface modeling)
        
    - 实体建模 (Solid modeling)
        
    - 直接编辑 (Direct editing)
        
    - 逆向工程 (Reverse engineering)
        

### 4. 商业扩展模块 (Commercial Extensions)

在核心开源模块下方，系统挂载了一系列商业化的高级应用模块。这些模块依赖内核进行计算，并将结果反馈回系统。包括：

- **SMRU**（通常指特定的特征识别或网格处理模块）
    
- **CNC, DFM**（计算机数控机加工特征、面向制造的设计分析）
    
- **DXF**（特定的二维图纸处理能力）
    
- **ALM**（增材制造/3D打印相关的几何处理）
    

### 5. 数据输出与前端展示 (Outputs & Web UI)

架构图右侧展示了系统处理后的输出能力：

- **多格式导出：** 可以将结果导出为多种格式，包括 3D 模型（STEP, BREP, IGES, STL, glTF）、2D 图像/矢量图（PNG, SVG, DXF）以及轻量化数据（JSON）。
    
- **Web UI：** 输出的数据（特别是轻量化的 glTF 和 JSON）可以直接驱动一个开源的 Web 用户界面，实现云端的 3D 可视化和交互。
    

### 6. 系统集成与调用 (Interfaces)

左下角标明了该内核的三种软件交付和调用形态：

- **SDK：** 软件开发工具包，供开发者二次开发。
    
- **App：** 独立的桌面端或服务端应用程序。
    
- **CLI：** 命令行接口，便于脚本自动化调用和批处理。
