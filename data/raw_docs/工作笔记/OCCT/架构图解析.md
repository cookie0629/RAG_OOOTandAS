### 1. 核心底座 (Foundation Layer)

架构图的最底层是所有其他模块的基础。

- **Foundation Classes (基础类)**
    
    - **定位：** 整个 OCCT 的基石，提供最基础的数据结构和系统服务。
        
    - **核心功能：**
        
        - **基础类型与集合：** 字符串 (Unicode)、数组、列表、哈希表等。
            
        - **数学与代数：** 基础线性代数（向量、矩阵运算、解线性方程组）。
            
        - **内存管理：** 提供智能指针 (`Handle` 机制) 实现对象的自动垃圾回收，以及专门的内存分配器。
            
        - **系统服务：** 异常处理机制、RTTI（运行时类型信息）、C++ 流封装、进度指示器。
            

### 2. 建模与几何核心 (Modeling Core)

架构图中央的拼图块，是处理 CAD 3D 几何与拓扑的灵魂模块。

- **Modeling Data (建模数据)**
    
    - **定位：** 定义和存储 3D 对象的 **边界表示法 (BRep)**。
        
    - **核心概念 (几何 vs 拓扑)：**
        
        - **几何 (Geometry)：** 物体的纯数学描述（如参数曲线、NURBS 曲面、点、向量的坐标和空间变换）。
            
        - **拓扑 (Topology)：** 将几何对象绑定在一起的数据结构，定义了实体间的关系。包括：`Vertex` (顶点) -> `Edge` (边) -> `Wire` (线框) -> `Face` (面) -> `Shell` (壳) -> `Solid` (实体) -> `Compound` (组合体)。
            
- **Modeling Algorithms (建模算法)**
    
    - **定位：** 提供对“建模数据”进行处理和生成的各种算法。
        
    - **核心功能：**
        
        - **底层工具：** 计算交点/交线、投影、形状细分、计算局部/全局属性（导数、质量惯性）。
            
        - **顶层 API (特征构建)：**
            
            - 图元创建：长方体、圆柱、圆锥、球体等。
                
            - 运动学建模：拉伸 (Prism)、旋转 (Revolve)、扫掠 (Pipe)、放样 (Loft)。
                
            - **布尔运算：** 交 (Common)、并 (Fuse)、差 (Cut)。
                
            - 局部修改：抽壳、倒角、拔模角。
                
- **Mesh (网格化)**
    
    - **定位：** 将 BRep 精确几何模型转换为用于渲染或分析的近似网格模型。
        
    - **核心功能：** 生成和存储三角形表面网格（Tessellation），是连接精确几何和可视化显示的桥梁。
        

### 3. 数据交互与修复 (Data Interoperability)

架构图左侧的拼图块，解决 OCCT 与外部 CAD 生态的交互问题。

- **Data Exchange (数据交换)**
    
    - **定位：** 读写外部 CAD 模型格式。
        
    - **支持格式：**
        
        - **标准/开源格式：** STEP, IGES, glTF, OBJ, VRML, STL 。
            
        - **XDE (扩展数据交换)：** 除了纯几何，还能解析颜色、图层、名称、材质等附加属性。
            
        - _(注：部分专有格式如 ACIS, Parasolid, JT 需高级商业组件支持)_。
            
- **Shape Healing (形状修复 - 内置于数据交换中)**
    
    - **定位：** 各个 CAD 系统的内核公差和标准不同，导入模型时常出现破面或几何错误。此模块专门负责“治病”。
        
    - **核心功能：**
        
        - **分析：** 检查边/线框的连通性、面朝向、公差冲突。
            
        - **修复：** 缝合破损线框、填补面与边之间的间隙 (Gap filling)。
            
        - **升级：** 统一几何类型（如将各种曲面统一转换为 B-Spline 曲面）。
            

### 4. 渲染与显示 (Visualization)

架构图右侧的独立支柱。

- **Visualization (可视化)**
    
    - **定位：** 将后台的 CAD 数据转化为屏幕上的 3D 图形。采用**数据与图形表示分离**的设计模式。
        
    - **核心功能：**
        
        - 基于 **OpenGL / OpenGL ES** 实现底层渲染 (`TKOpenGl`)。
            
        - 高层交互 API (`AIS` - Application Interactive Services)，支持对象的交互式高亮和选择。
            
        - 高级特性：PBR 物理渲染材质、动态剪裁平面 (Clipping planes)、光线追踪 (Ray tracing)、视锥剔除以优化性能。
            

### 5. 顶层应用框架 (Application Layer)

架构图中位于各核心模块之上，GUI 之下。

- **OCAF (Open CASCADE Application Framework)**
    
    - **定位：** 帮助开发者快速搭建复杂的 CAD 软件，而不是仅仅写一个算法脚本。
        
    - **核心概念：Application / Document 范式**。
        
    - **核心功能：**
        
        - 通过 **Reference Keys (参考键)** 而非直接操作 Shape 来组织数据树。
            
        - 处理父子参数化依赖关系（参数化建模基础）。
            
        - 开箱即用的撤销/重做 (Undo/Redo)、复制/粘贴。
            
        - 数据持久化（保存和打开项目文件）。
            

### 6. 开发辅助工具 (Development Tools)

架构图最左侧的支柱。

- **Draw Test Harness (Draw 测试线束)**
    
    - **定位：** 一个基于 Tcl 脚本语言的命令行交互环境。
        
    - **作用：** 在编写复杂的 C++ UI 程序之前，可以直接在命令行输入命令来测试各种底层 API（如生成形体、测试布尔运算、查看数据交换结果），并带有简单的 2D/3D 查看器。这是学习和验证 OCCT 算法最高效的沙盒。
